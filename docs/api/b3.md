# Loomio API /api/b3

`/api/b3` is for server-level operations. Use `/api/b2` for user-oriented actions performed as a Loomio user account.

## Authentication

Set `B3_API_KEY` to a secret longer than 16 characters.

Send the key as a bearer token:

```bash
curl -H 'Authorization: Bearer secret_not_shown' {{root_url}}api/b3/users
```

The older `b3_api_key` request param is still accepted for compatibility.

## User Object

User responses use this shape:

```json
{
  "id": 123,
  "name": "Ada Lovelace",
  "username": "ada",
  "email": "ada@example.org",
  "active": true,
  "deactivated_at": null,
  "identities": [
    {
      "id": 456,
      "identity_type": "oauth",
      "uid": "external-123",
      "email": "ada@example.org",
      "name": "Ada Lovelace"
    }
  ]
}
```

## List Users

```bash
curl -H 'Authorization: Bearer secret_not_shown' {{root_url}}api/b3/users
```

Returns:

```json
{
  "users": []
}
```

## Show User

By Loomio user ID:

```bash
curl -H 'Authorization: Bearer secret_not_shown' {{root_url}}api/b3/users/123
```

By external identity:

```bash
curl -H 'Authorization: Bearer secret_not_shown' {{root_url}}api/b3/users/identity/oauth/external-123
```

Returns:

```json
{
  "user": {}
}
```

## Update User

By Loomio user ID:

```bash
curl -X PATCH \
  -H 'Authorization: Bearer secret_not_shown' \
  -H 'Content-Type: application/json' \
  -d '{"user":{"name":"Ada Lovelace","username":"ada","email":"ada@example.org"}}' \
  {{root_url}}api/b3/users/123
```

By external identity:

```bash
curl -X PATCH \
  -H 'Authorization: Bearer secret_not_shown' \
  -H 'Content-Type: application/json' \
  -d '{"user":{"name":"Ada Lovelace","username":"ada","email":"ada@example.org"}}' \
  {{root_url}}api/b3/users/identity/oauth/external-123
```

Only these fields can be updated:

| Field | Description |
| --- | --- |
| `name` | Display name |
| `username` | Loomio username |
| `email` | Email address |

Returns the updated user:

```json
{
  "user": {}
}
```

## Deactivate User

By Loomio user ID:

```bash
curl -X POST -H 'Authorization: Bearer secret_not_shown' {{root_url}}api/b3/users/123/deactivate
```

By external identity:

```bash
curl -X POST -H 'Authorization: Bearer secret_not_shown' {{root_url}}api/b3/users/identity/oauth/external-123/deactivate
```

Returns:

```json
{
  "success": true,
  "user": {}
}
```

## Delete User

Delete uses Loomio's existing user redaction behavior. It does not raw-delete database rows.

By Loomio user ID:

```bash
curl -X DELETE -H 'Authorization: Bearer secret_not_shown' {{root_url}}api/b3/users/123
```

By external identity:

```bash
curl -X DELETE -H 'Authorization: Bearer secret_not_shown' {{root_url}}api/b3/users/identity/oauth/external-123
```

Returns:

```json
{
  "success": true
}
```

## SSO Profile Sync Settings

Use these settings when another system manages Loomio profile fields.

```env
LOOMIO_DISABLE_EDIT_USER_PROFILE=1
LOOMIO_SSO_UPDATE_USER_PROFILE_ON_LOGIN=0
```

`LOOMIO_DISABLE_EDIT_USER_PROFILE=1` prevents users from editing these fields themselves:

| Field | Notes |
| --- | --- |
| `name` | Managed by external sync |
| `username` | Managed by external sync |
| `email` | Managed by external sync |
| `avatar_kind` / `uploaded_avatar` | Managed by external sync |

Users can still edit Loomio-local fields such as `short_bio` and `location`.

`LOOMIO_SSO_UPDATE_USER_PROFILE_ON_LOGIN=true` updates `name` and `email` from SSO login data. Leave it unset or set it to `0` when an external sync script should be the only source of those updates.

`LOOMIO_SSO_FORCE_USER_ATTRS` still works for existing installs. It both disables user edits and updates `name` and `email` on SSO login.
