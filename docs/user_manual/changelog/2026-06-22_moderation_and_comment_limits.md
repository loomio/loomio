# Moderation and comment limits

Admins now have more control over comment length and vote-reason visibility.

## Comment length limits

Threads and templates can now define a comment length limit.

When a limit is set, comments must stay within the configured character count. If a comment is too long, Loomio shows a clear validation message before it is posted.

This is useful for groups that want concise input, structured check-ins, or lightweight decision-making formats.

## Templates can include limits

Discussion and poll templates can include a comment length limit, so repeated processes can carry the same expectation each time they are used.

For example, a template can encourage short updates or tightly scoped feedback without admins having to set the limit manually on every new thread.

## Vote reason redaction

Admins can now redact vote reasons.

When a vote reason is redacted:

- the vote remains in the poll results
- the reason text is hidden
- attachments on the reason are hidden
- the redaction can be restored if needed

This gives admins a way to moderate sensitive or inappropriate vote explanations without changing the actual vote.

## File upload size configuration

Installations can now configure the maximum upload size. This may affect the largest files people can attach, depending on how the Loomio site is configured.
